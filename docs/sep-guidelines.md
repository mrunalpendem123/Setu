# SEP Guidelines (Spec Enhancement Proposal)

Use an SEP for significant changes to APIs, specs, or protocol behavior.

## When to use SEP
- Changes to checkout/payment schemas
- Versioning updates
- New extensions or capabilities

## SEP Template
1. Summary
2. Motivation
3. Proposal
4. Backwards compatibility
5. Security considerations
6. Migration plan
7. Open questions

## Process
- Open an issue and attach the SEP document
- Maintainer review
- Accepted SEPs are moved into `rfc/`
