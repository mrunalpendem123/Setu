# Governance

This project uses a lightweight maintainer-driven governance model.

## Roles
- **Maintainers**: approve PRs, manage releases, and enforce policy.
- **Contributors**: submit PRs and propose changes via issues.

## Decision Making
- Minor changes: maintainer approval.
- Major changes: use the SEP process (see `docs/sep-guidelines.md`).

## Contacts
See `MAINTAINERS.md`.
