# Contributing

Thanks for your interest in contributing.

## How to Contribute
1. Fork the repo and create a branch.
2. Make your changes with clear commit messages.
3. Open a PR with a summary and testing notes.

## Branching Model
- `main` is always deployable.
- Feature branches: `feature/<name>`.

## Pull Requests
- Link issues if applicable.
- Include screenshots or logs when relevant.
- Ensure docs are updated if you change APIs.

## Tests
- Run unit tests if present.
- For now, `scripts/validate_specs.py` is the required check.

## Code Style
- Python: PEP8 (black/ruff recommended).
- Rust: `cargo fmt` and `cargo clippy`.

## Security
Please report security issues privately to the maintainers.
